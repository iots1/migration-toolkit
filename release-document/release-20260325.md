## Detailed Fix Log

### 1. `services/transformers.py` — 3 High-Severity Fixes

#### Fix #1: NaN-to-String Data Corruption (Lines 81-97)
**Severity:** HIGH | **Impact:** Data Corruption in all migrations
**Bug:** `series.astype(str)` converts `NaN`/`None` → literal string `"nan"`/`"None"`, corrupting NULL values in target DB

**Before:**
```python
if transformer_name == "TRIM":
    return series.astype(str).str.strip()

if transformer_name == "UPPER_TRIM":
    return series.astype(str).str.strip().str.upper()

if transformer_name == "CLEAN_SPACES":
    return series.astype(str).str.replace(r'\s+', ' ', regex=True).str.strip()

if transformer_name == "TO_NUMBER":
    return series.astype(str).str.replace(r'\D', '', regex=True)

if transformer_name == "REPLACE_EMPTY_WITH_NULL":
    return series.where(series.astype(str).str.strip().ne('') & series.astype(str).str.strip().ne('None'), other=np.nan)
```

**After:**
```python
# Preserve NaN/None using series.where(series.isna(), ...)
if transformer_name == "TRIM":
    return series.where(series.isna(), series.astype(str).str.strip())

if transformer_name == "UPPER_TRIM":
    return series.where(series.isna(), series.astype(str).str.strip().str.upper())

if transformer_name == "CLEAN_SPACES":
    return series.where(series.isna(), series.astype(str).str.replace(r'\s+', ' ', regex=True).str.strip())

if transformer_name == "TO_NUMBER":
    return series.where(series.isna(), series.astype(str).str.replace(r'\D', '', regex=True))

if transformer_name == "REPLACE_EMPTY_WITH_NULL":
    return series.where(series.notna() & series.astype(str).str.strip().ne(''), other=np.nan)
```

**Result:** NULL/NaN values in source DB are now preserved as NULL in target DB (not written as strings)

#### Fix #2: Buddhist Year Detection Threshold (Line 170-172)
**Severity:** MEDIUM | **Impact:** Wrong dates written to DB
**Bug:** Threshold `year_val > 2000` incorrectly treats Gregorian years 2001-2399 as Buddhist Era

**Before:**
```python
year_val = int(y)
iso_year = year_val - 543 if year_val > 2000 else year_val
# Problem: CE 2024 → 2024 - 543 = 1481 (wrong!)
```

**After:**
```python
year_val = int(y)
# BE years in Thailand are ~2500+; threshold >2400 avoids
# misidentifying Gregorian years (2001-2399) as Buddhist Era.
iso_year = year_val - 543 if year_val > 2400 else year_val
# Now: CE 2024 → 2024 (not converted)
#      BE 2567 → 2567 - 543 = 2024 (correct)
```

**Result:** Date conversions are now accurate for both Gregorian and Buddhist calendar inputs

#### Fix #3: Unvalidated Date Fallback (Lines 189-197)
**Severity:** MEDIUM | **Impact:** Invalid dates written to DB
**Bug:** Manual date parsing fallback doesn't validate day (1-31), month (1-12), year > 0

**Before:**
```python
try:
    parts = re.split(r'[-/]', date_str.strip())
    if len(parts) == 3:
        d, m, y = parts
        return f"{y}-{m.zfill(2)}-{d.zfill(2)}"  # No validation!
except:
    return None
```

**After:**
```python
try:
    parts = re.split(r'[-/]', date_str.strip())
    if len(parts) == 3:
        d, m, y = parts
        d_val, m_val, y_val = int(d), int(m), int(y)
        if not (1 <= m_val <= 12 and 1 <= d_val <= 31 and y_val > 0):
            return None  # Invalid date → NULL
        return f"{y_val:04d}-{m_val:02d}-{d_val:02d}"
except (ValueError, AttributeError):
    return None
```

**Result:** Invalid dates (e.g., "99-99-99") now return NULL instead of being written as invalid strings

#### Fix #5: Bare Exception Handling (Lines 20, 46)
**Severity:** LOW | **Impact:** Silent failures hide unexpected errors
**Bug:** `except:` catches all exceptions including `KeyboardInterrupt`, `SystemExit`

**Before:**
```python
if validator_name == "POSITIVE_NUMBER":
    try:
        return (float(value) > 0, "Must be > 0")
    except:  # Catches everything
        return (False, "Not a number")

if validator_name == "VALID_DATE":
```

**After:**
```python
if validator_name == "POSITIVE_NUMBER":
    try:
        return (float(value) > 0, "Must be > 0")
    except (ValueError, TypeError):  # Only expected exceptions
        return (False, "Not a number")

if validator_name == "VALID_DATE":
    try:
        datetime.strptime(value, '%Y-%m-%d')
        return (True, "")
    except (ValueError, TypeError):  # Only expected exceptions
        return (False, "Invalid Date (YYYY-MM-DD)")
```

**Result:** Unexpected errors (system-level) now propagate instead of being silently swallowed

### 3. `utils/helpers.py` — 2 Fixes

#### Fix #6: pd.isna() Check Order (Lines 7-11)
**Severity:** MEDIUM | **Impact:** Crash on array-like inputs
**Bug:** `pd.isna(val)` before `val is None` raises `ValueError` when `val` is a Series/list

**Before:**
```python
def safe_str(val):
    if pd.isna(val) or val is None: return ""  # pd.isna crashes on arrays
    return str(val).strip()
```

**After:**
```python
def safe_str(val):
    if val is None: return ""
    try:
        if pd.isna(val): return ""
    except (ValueError, TypeError):
        pass  # pd.isna fails on some types
    return str(val).strip()
```

**Result:** Function now safely handles scalar values without crashes

#### Fix #7: get_report_folders Returns Files Too (Lines 34-36)
**Severity:** LOW | **Impact:** Non-directory files treated as report folders
**Bug:** `glob.glob()` returns both files and directories without filtering

**Before:**
```python
def get_report_folders():
    if not os.path.exists(MIGRATION_REPORT_DIR): return []
    folders = glob.glob(os.path.join(MIGRATION_REPORT_DIR, "*"))  # Files included!
    folders.sort(reverse=True)
    return folders
```

**After:**
```python
def get_report_folders():
    if not os.path.exists(MIGRATION_REPORT_DIR): return []
    folders = [f for f in glob.glob(os.path.join(MIGRATION_REPORT_DIR, "*")) if os.path.isdir(f)]
    folders.sort(reverse=True)
    return folders
```

**Result:** Only directories are returned, preventing stray files from being treated as report folders

### 4. `utils/ui_components.py` — 1 Fix

#### Fix #8: Dialog Confirm Has No Error Handling or Closure (Lines 68-71)
**Severity:** HIGH | **Impact:** Silent failures, UI frozen after action
**Bug:** `on_confirm_func()` has no try/catch, and dialog never explicitly closes

**Before:**
```python
with c_confirm:
    if st.button(confirm_label, type="primary", use_container_width=True):
        on_confirm_func(*args, **kwargs)  # No error handling, no st.rerun()!
```

**After:**
```python

    if st.button(confirm_label, type="primary", use_container_width=True):
        try:
            on_confirm_func(*args, **kwargs)
            st.rerun()  # Close dialog after success
        except Exception as e:
            st.error(f"Error: {e}")  # Show error to user
```

**Result:** Dialog now closes after successful action, and errors are displayed instead of causing silent crashes

### 5. `services/db_connector.py` — 3 Critical SQL Injection Fixes

#### Fix #9-11: SQL Injection in All Inspection Functions
**Severity:** CRITICAL | **Impact:** Arbitrary SQL execution possible
**Bug:** Schema, table, and column names interpolated directly into SQL strings without validation

**Added Safety Function (Line 8-17):**
```python
def _safe_id(name: str) -> str:
    """Validate DB identifier (table/schema/column) to prevent SQL injection.
    Allows alphanumeric, underscore, hyphen, dot, and spaces (for MSSQL schemas).
    Raises ValueError on suspicious input.
    """
    if not isinstance(name, str) or not name.strip():
        raise ValueError(f"Invalid identifier: {name!r}")
    if not re.match(r'^[a-zA-Z0-9_\-\. ]+$', name):
        raise ValueError(f"Unsafe identifier rejected: {name!r}")
    return name
```

**Applied to Functions:**

1. **`get_tables_from_datasource()` (Lines 233-234)**
   - Before: `WHERE TABLE_SCHEMA = '{schema_filter}'` ← raw user input
   - After: `schema_filter = _safe_id(schema) if schema else 'dbo'` ✓

2. **`get_columns_from_table()` (Lines 254-256)**
   - Before: `WHERE TABLE_NAME = '{table_name}'` ← raw user input
   - After: `safe_table = _safe_id(table_name)` ✓

3. **`get_foreign_keys()` (Lines 281-282)**
   - Before: `WHERE REFERENCED_TABLE_SCHEMA = '{db_name}'` ← raw user input
   - After: `safe_db = _safe_id(db_name)` ✓

4. **`get_table_sample_data()` (Lines 356-365)**
   - Before: `SELECT TOP {limit} * FROM {table_ref}` ← unsanitized identifiers
   - After: `safe_table = _safe_id(table_name); limit = int(limit)` ✓

5. **`get_column_sample_values()` (Lines 383-398)**
   - Before: `SELECT DISTINCT [{column_name}] FROM {table_ref}` ← unsafe
   - After: `safe_col = _safe_id(column_name); safe_table = _safe_id(table_name)` ✓

**Impact:** Prevents attacks like `schema="'; DROP TABLE patients; --"` from executing arbitrary SQL
